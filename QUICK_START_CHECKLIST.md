# LOCAL SEO WEBSITE QUICK-START CHECKLIST
## Daily/Weekly Task List for Building Local SEO Sites

---

## WEEK 1: FOUNDATION (Days 1-7)

### Day 1: Research & Planning
- [ ] Competitor analysis (top 5)
- [ ] Keyword research (SEMrush + Ubersuggest)
  - [ ] 5 primary keywords
  - [ ] 10 secondary keywords
  - [ ] 20 long-tail keywords
- [ ] URL architecture map
- [ ] Content strategy document

### Day 2: Setup
- [ ] Register domain
- [ ] Set up hosting (Vercel/Netlify)
- [ ] Configure HTTPS
- [ ] Set up Git repo
- [ ] Create folder structure
- [ ] Set up Google Analytics
- [ ] Set up Google Search Console

### Day 3: Design System
- [ ] Define color palette (save in CSS variables)
- [ ] Select fonts (Google Fonts)
- [ ] Create base styles.css
- [ ] Build button components
- [ ] Build card components
- [ ] Build form components

### Day 4: Core Templates
- [ ] Header component (sticky, mobile-responsive)
- [ ] Footer component
- [ ] Homepage template structure
- [ ] Service page template
- [ ] Location page template
- [ ] Blog post template

### Day 5: Technical Setup
- [ ] robots.txt
- [ ] vercel.json / .htaccess
- [ ] sitemap.xml structure
- [ ] site.webmanifest
- [ ] 404.html
- [ ] Schema templates (JSON-LD)

### Day 6: Google Business Profile
- [ ] Claim listing
- [ ] Complete all sections
- [ ] Add services
- [ ] Upload photos
- [ ] Set up Q&A
- [ ] Connect website

### Day 7: Review
- [ ] Check all setup is working
- [ ] Test mobile responsiveness
- [ ] Validate schema markup
- [ ] Prepare content briefs for Week 2

## WEEK 3: SEO IMPLEMENTATION (Days 15-21)

### Day 15: On-Page SEO (Part 1)
- [ ] Add titles to all pages (50-60 chars)
- [ ] Add meta descriptions (150-155 chars)
- [ ] Add canonical tags
- [ ] Verify H1 on every page
- [ ] Check heading hierarchy

### Day 16: On-Page SEO (Part 2)
- [ ] Add Open Graph tags
- [ ] Add Twitter Card tags
- [ ] Add author meta tags
- [ ] Add publish dates
- [ ] Add breadcrumb schema

### Day 17: Schema Markup
- [ ] Organization schema (all pages)
- [ ] LocalBusiness schema (all pages)
- [ ] Service schema (service pages)
- [ ] BlogPosting schema (blog posts)
- [ ] FAQPage schema (all FAQs)
- [ ] BreadcrumbList schema

### Day 18: Internal Linking
- [ ] Link homepage to services
- [ ] Link services to locations
- [ ] Link locations to services
- [ ] Link blog to services
- [ ] Create footer links
- [ ] Add breadcrumb navigation

### Day 19: Image Optimization
- [ ] Compress all images
- [ ] Convert to WebP
- [ ] Add alt text to all images
- [ ] Add width/height attributes
- [ ] Implement lazy loading

### Day 20: Technical SEO
- [ ] Preconnect to fonts.googleapis.com
- [ ] Inline critical CSS
- [ ] Defer non-critical JS
- [ ] Add security headers
- [ ] Test page speed

### Day 21: Local SEO
- [ ] Add NAP to all pages (consistent format)
- [ ] Add Google Maps embed
- [ ] Create location schema with geo
- [ ] Add local landmarks to content
- [ ] Submit to 10 directories

---

## WEEK 4: QA & LAUNCH (Days 22-30)

### Day 22: Screaming Frog Audit
- [ ] Crawl entire site
- [ ] Fix all 404 errors
- [ ] Fix duplicate titles
- [ ] Fix missing alt text
- [ ] Fix broken links
- [ ] Fix heading hierarchy issues

### Day 23: SEMrush Audit
- [ ] Run site audit
- [ ] Fix all errors
- [ ] Fix all warnings
- [ ] Check Core Web Vitals
- [ ] Verify mobile-friendliness

### Day 24: Ubersuggest Audit
- [ ] Run site audit
- [ ] Check SEO score
- [ ] Identify backlink opportunities
- [ ] Check content gaps

### Day 25: Manual Testing
- [ ] Test on iPhone
- [ ] Test on Android
- [ ] Test on iPad
- [ ] Test on Desktop
- [ ] Test all forms
- [ ] Test all CTAs
- [ ] Test phone links

### Day 26: Content Review
- [ ] Grammar check (Grammarly)
- [ ] Readability check (Hemingway)
- [ ] Keyword density check
- [ ] Internal link check
- [ ] Remove placeholder text

### Day 27: Pre-Launch
- [ ] Generate XML sitemaps
- [ ] Submit to Google Search Console
- [ ] Submit to Bing Webmaster
- [ ] Set up rank tracking (SEMrush)
- [ ] Configure alerts

### Day 28: Soft Launch
- [ ] Launch site (password protected if needed)
- [ ] Share with team/client
- [ ] Collect feedback
- [ ] Fix any issues
- [ ] Prepare launch announcement

### Day 29: Official Launch
- [ ] Remove password protection
- [ ] Announce on social media
- [ ] Send email announcement
- [ ] Monitor traffic
- [ ] Monitor for errors

### Day 30: Post-Launch
- [ ] Check indexing status
- [ ] Verify Analytics working
- [ ] Check Search Console for errors
- [ ] Respond to any issues
- [ ] Plan Month 2 content

## EMERGENCY FIXES (If Something Breaks)

### Site Down
1. [ ] Check hosting status
2. [ ] Check domain expiry
3. [ ] Check DNS settings
4. [ ] Rollback last Git commit
5. [ ] Contact hosting support

### Ranking Drops
1. [ ] Check Google Search Console for penalties
2. [ ] Check for algorithm updates
3. [ ] Audit for technical issues
4. [ ] Check competitor activity
5. [ ] Review recent changes

### Traffic Drops
1. [ ] Check Analytics for source
2. [ ] Check Search Console for indexing issues
3. [ ] Check for broken pages
4. [ ] Check Core Web Vitals
5. [ ] Review content quality
